## Codebase Patterns
- Frontend typecheck: `cd frontend && npx tsc --noEmit`
- Backend tests: `uv run python -m pytest` (not `uv run pytest` — the latter has import issues)
- ConversationTimeline uses `expandKey` state bump to force TurnCard re-mount when toggling expand/collapse all
- TurnCard accepts `defaultExpanded` prop, initializes internal `expanded` state from it
- `TruncatableContent` component uses CSS `line-clamp-3` + ResizeObserver to detect overflow and show View/Hide toggle
- ToolInputDetail already has its own view/hide toggle (content hidden by default), so no extra truncation needed there
- `_extract_memory_refs()` scans JSONL for `memory/<filename>.md` patterns — regex is `memory/([A-Za-z0-9_.\-]+\.md)`
- `load_memory_files()` now filters by session-referenced files; both REST and WS endpoints use it so both are filtered
- MemoryPanel component is in `frontend/src/components/MemorySidebar.tsx` (not `session/`), rendered via nested route `<Outlet />`
- `_find_project_dir()` resolves a decoded project path back to the Claude project directory name by scanning `~/.claude/projects/`
- `load_all_memory_files()` loads all .md files from a project's memory dir without session-scoping (used by project-level endpoint)
- Skills have two file formats: `{name}.md` (flat file) and `{name}/SKILL.md` (directory-based)
- `installed_plugins.json` at `~/.claude/plugins/` maps plugin names to install paths; each plugin's skills are in `{installPath}/skills/`
- Skill deduplication: load lowest priority first (plugin), then overwrite with higher (user, project) — simple dict key replacement
