## Codebase Patterns
- Python source lives in `src/clawhawk/` with hatchling build backend
- Use `uv run pyright src/clawhawk/` for typechecking
- CLI entry point is `clawhawk.app:main`, also runnable via `python -m clawhawk`
- Dev dependencies go in `[dependency-groups] dev` section of pyproject.toml
- All public Pydantic models use `ConfigDict(alias_generator=to_camel, populate_by_name=True)` for camelCase JSON
- JSONL top-level fields are camelCase but `usage` sub-fields are snake_case — only `Message` needs alias_generator
- For stats parsing, use raw `json.loads` instead of Pydantic models for performance (only need type, timestamp, usage fields)
- Python `date.weekday()` returns 0=Monday (unlike Go where Sunday=0), so Monday-based week start = `now - timedelta(days=now.weekday())`
- WebSocket/API routes must be registered BEFORE the SPA catch-all `/{full_path:path}` route in app.py
- `_PROJECT_ROOT = Path(__file__).resolve().parent.parent.parent` goes from `src/clawhawk/app.py` to project root
- Use `asyncio.to_thread()` to call sync/blocking functions (file I/O, os.kill) from async handlers
- Frontend uses bun 1.3.5 as package manager; HeroUI v3 is @heroui/react + @heroui/styles (use @latest, not @beta)
- CSS import order for HeroUI: `@import "tailwindcss"` first, then `@import "@heroui/styles"`
- HeroUI v3 Card uses compound component pattern: Card.Header, Card.Content, Card.Footer (not separate CardHeader/CardBody/CardFooter)
- WebSocket must use `ws.send_text()` not `ws.send_bytes()` — browser `JSON.parse(event.data)` fails silently on Blob data
- HeroUI Meter component accepts color prop ('default'|'warning'|'danger') and has Meter.Track, Meter.Fill, Meter.Output subcomponents
- Tooltip uses compound pattern: Tooltip > Tooltip.Trigger > Tooltip.Content (with Tooltip.Arrow)
- HeroUI Button variant options: 'primary'|'secondary'|'tertiary'|'danger'|'danger-soft'|'ghost'|'outline' (NOT 'bordered')
- Tailwind v4 responsive grid: use `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 2xl:grid-cols-4` for 1-4 column responsive layouts
- HeroUI Chip variant options: 'primary'|'secondary'|'tertiary'|'soft' (NOT 'dot' — use a manual span element inside Chip for dot indicators)
- HeroUI Chip does NOT have `startContent` prop — place decorative elements as children instead
